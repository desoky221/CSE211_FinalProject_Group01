================================================================================
REQUIRED DOCUMENTATION FILES
================================================================================

1. Website_Map.pdf
   - Description: Visual representation of the website structure and navigation
   - Contents: Site map showing all pages, their relationships, and navigation flow
   - Purpose: Demonstrates information architecture and site organization
   - Requirements: Must show 7-10 web pages as per project specifications

2. Validation_Report.pdf
   - Description: Comprehensive validation results for all code files
   - Contents: 
     * HTML5 validation screenshots/results for all HTML files
     * CSS3 validation screenshots/results for all CSS files
     * JavaScript validation results (if applicable)
     * PHP validation results (if bonus part implemented)
   - Purpose: Demonstrates code quality and adherence to W3C standards
   - Requirements: All HTML, CSS, JavaScript, and PHP files must be validated
   - Note: Validation screenshots should be saved in the images folder as per
           specifications (Section 4.1, item 6)

3. Browser_Testing_Report.pdf
   - Description: Cross-browser compatibility testing results
   - Contents:
     * Testing results for Microsoft Edge (current version)
     * Testing results for Firefox (current version)
     * Testing results for Google Chrome (current version)
     * Testing results for Opera (current version)
     * Testing results for Safari (Mac and Windows)
     * Testing results for mobile browsers (tablets and smartphones)
   - Purpose: Demonstrates cross-browser compatibility as required
   - Requirements: Website must render correctly in all specified browsers
   - Note: Section 4.1, item 10 specifies browser requirements

4. AI_Tools_Usage_Documentation.pdf
   - Description: Documentation of AI assistance used in the project
   - Contents:
     * List of AI tools used (ChatGPT, Copilot, Claude, etc.)
     * Description of AI-assisted components
     * Specific code sections or content created with AI assistance
     * Learning outcomes from AI usage
   - Purpose: Acknowledgment of AI assistance as required by course guidelines
   - Requirements: Must be included in project report Appendix A
   - Note: Section 1.4 and Section 7 specify AI assistance acknowledgment

5. HTML5_Validation_Report.pdf (Additional)
   - Description: Detailed HTML5 validation report
   - Contents: Comprehensive HTML validation results
   - Purpose: Additional validation documentation
   - Note: This is supplementary to Validation_Report.pdf
